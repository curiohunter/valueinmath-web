## 분석 방법론

### 1. 사이클 기준

**N월분 사이클** = (N-1)월 25일 ~ N월 24일

| 분석월 | 사이클 기간 | 수입 기준 | 비용 기준 |
|--------|------------|----------|----------|
| 1월 | 12/25 ~ 1/24 | 해당 기간 입금액 | 1월 발생 비용 |
| 2월 | 1/25 ~ 2/24 | 해당 기간 입금액 | 2월 발생 비용 |

### 2. 발생주의 회계 (중요!)

**인건비는 발생 시점 기준으로 분류:**

| 지급일 | 해당 월 | 비용 귀속 |
|--------|--------|----------|
| 1월 10일 지급 | 12월분 인건비 | **12월 비용** |
| 2월 10일 지급 | 1월분 인건비 | **1월 비용** |
| 3월 10일 지급 | 2월분 인건비 | **2월 비용** |

**주의**: 1월 사이클 분석 시:
- 은행 내역의 1/10 인건비 → **12월 비용** (제외)
- 1월 비용 인건비 → **2/10 지급 예정액** (미지급, 예상치 사용)

### 3. 현금주의 vs 발생주의

| 항목 | 기준 | 설명 |
|------|------|------|
| 수입 | **현금주의** | 실제 입금된 금액 기준 |
| 비용 | **발생주의** | 비용 발생 시점 기준 |
| 인건비 | **발생주의** | N월분 = (N+1)월 10일 지급 예정액 |
| 임대료 | **현금주의** | 실제 지급 시점 (21~23일) |

## 분석 실행 절차

### Step 1: 파일 읽기 및 파싱
```
1. 은행 내역 파일 읽기 (세미콜론 구분)
2. 날짜, 출금, 입금, 내용, 잔액 파싱
3. 숫자에서 콤마 제거
```

### Step 2: 사이클 필터링
```
1. 분석월 확인 (예: 2026-01)
2. 사이클 기간 계산 (12/25 ~ 1/24)
3. 해당 기간 거래만 필터링
```

### Step 3: 거래 분류
```
1. 입금 거래 → 카드사별/현금 분류
2. 출금 거래 → 카테고리별 분류
3. 인건비는 발생월 기준으로 재분류
```

### Step 4: 집계 및 계산
```
1. 총 수입 = 카드 정산 + 현금/이체
2. 총 지출 = 인건비(발생주의) + 고정비 + 기타
3. 영업이익 = 총 수입 - 총 지출
4. 이익률 = 영업이익 / 총 수입 × 100
```

### Step 5: 현금 흐름표 작성
```
1. 기초 잔액 (사이클 시작일)
2. 영업활동 현금흐름 = 수입 - 지출(현금주의)
3. 재무활동 = 대출상환
4. 기말 잔액 (사이클 종료일)
5. 미래 현금 예측 (미지급/미수금)
```

### Step 6: 리포트 생성
```
docs/valueinmath_docs/재무/YYMMDD_재무_YYYY년M월-재무분석.md
```

## 주의사항

1. **인건비 발생주의**: 은행 내역의 인건비 지급일이 아닌, 비용 발생월 기준
2. **사이클 기준**: 수강료는 전월 25일~당월 24일 입금분
3. **특강**: 1-2월만 해당, 담당 선생님 매출의 50%가 인건비
4. **미지급 비용**: 현금 흐름 예측 시 반드시 포함
5. **카드 정산**: D+2~3 지연 고려
